### 7.3 Best Practices

**Why follow these**: Good structure and clear descriptions make the agent easier to maintain and debug. When you add more flows or hand the project to someone else, consistent naming and testing habits keep things from breaking.

- **Organizing**: One flow per file; descriptive names; group related flows in folders.
- **Responses**: Clear and concise; use variations; include context for next steps.
- **Flow descriptions**: Be specific; use action verbs; include context about what information the flow provides.
- **Testing**: Test each flow individually; try different phrasings; run a full conversation; check logs if something fails.
